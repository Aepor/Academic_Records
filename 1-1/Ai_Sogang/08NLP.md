# 텍스트 분석
비정형 데이터인 텍스트를 피처 형태로 추출하여 의미 있는 값을 부여하는 것이 중요함.
이를 Feature Vectorization (피처 벡터화) 라고 부름.
예전 : BOW, Word2Vec
현재(2026) : BERT(문맥 고려), API기반 임베딩

1. 텍스트 전처리
텍스트를 피처로 만들기 전, 대/소문자 변경, 특수문자 삭제, 단어의 토큰화, 의미 없는 단어 제거, 어근 추출 등의 텍스트 정규화 작업을 수행하는 것.

2. 피처 벡터화/추출
사전 준비 작업으로 가공된 텍스트에서 피처를 추출하고 벡터 값을 할당하는 것. (BOw or Word2Vec)

3. ML모델 수립 및 학습/예측/평가
피처 벡터화된 데이터 세트에 ML모델을 적용해 학습/예측 및 평가 수행.

* 데이터 전처리
클렌징 : 텍스트에서 분석에 방해가 되는 문자, 기호 등을 사전에 제거
토큰화 : 문장 토큰화(문장 단위로 쪼갬.) 와 단어 토큰화(단어 단위로 쪼갬.)로 나뉨. 긴 문장을 쪼개는 과정
n-gram : n개의 단어를 하나의 토큰화 단위로 분리해 내는 것. 
Stop Word : 의미 없는 단어 지칭 (is, the, a , will 등) 언어별로 목록화돼 있음.
Stemming, Lemmatization : 단어의 원형을 찾음. 품사를 입력받는 Lemmatization이 더 정교하지만, 시간이 오래 걸림. ex.(working -> work)

* 피처 벡터화
BOW (Bag of Words)
문맥, 순서를 무시하고 일괄적으로 빈도 값을 부여해 피처를 추출함.
장점: 쉽고 빠르고, 단어 발생 횟수에 기반하고 있으나, 문서 특징이 잘 나타낼 수 있음.
단점: 문맥적 의미 반영이 부족하고, 많은 자료에서 벡터를 추출 했을 때 희소 행렬 문제가 발생.  희소행렬: 대부분의 값이 0으로 채워지는 행렬 (Sparse Matrix)

* Word2Vec
인공신경망을 통해 주변 맥락을 고려하여 단어를 벡터로 표현, 단어간의 유사도와 관계를 파악할 수 있음. 
단점 : 학습 비용 높음, 사전에 없는 단어에 취약함, 문맥적 의미 반영이 부족함. ex. 먹는 배와 타는 배 구분 힘듦.

* 희소행렬 변환 방식 (Scipy이용)
희소행렬은 불필요한 0 값이 메모리 공간에 할당되어 메모리 공간을 소모시키기에, 변환해놓는 과정이 필요.

COO : 0이 아닌 데이터만 데이터 배열에 저장하고 그 데이터의 행과 열의 위치를 다시 별도의 배열에 저장함.
ex. [[3, 0, 1], [0,2,0]] -> [3,1,2], (row, col) : (0,0), (0,2), (1,1) -> [0,0,1],[0,2,1]  => 순차적으로 생각.

CSR : 0이 아닌 데이터 값의 행과 행 위치를 나타내어 배열에 저장하는 건 COO와 같지만, 행속의 데이터가 어디부터 달라지는지에 대한 배열 또한 만들어서 연산 시간을 줄임.
[0,0,1] 같은경우 0이 인덱스1까지 1이 2부터 시작하기에 [0,2]를 가짐.
