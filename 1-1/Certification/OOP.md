#객체지향프로그래밍
*대표적으로 JAVA, C++이 존재.
객체와 객체들이 모여 프로그램을 구성함.
주요 구성요소에는 객체, 클래스, 메시지가 있음.
**JAVA : 분산 네트워크 환경에 적용 가능, 멀티스레드 기능 제공, **운영체제 및 하드웨어에 독립적
*기계어로 바로 컴파일되는 게 아니라 bytecode로 컴파일됨. 이를 운영체제에 맞는 JVM(Java Virtual Machine)이 실행해줌.
즉, 운영체제마다 다시 컴파일할 필요가 없음. 

*장단점
장점 : 
1. 상속을 통한 재사용과 시스템의 확장 용이
2. 자연적인 모델링 : 분석과 설계가 쉽고 효율적임.
3. 대형 프로그램 작성의 용이성
4. 소프트웨어 개발 및 유지보수가 용이.

단점 :
1. 구현 시 처리 시간이 지연
2. 정형화된 분석 및 설계 방법 X

*Object
Attribute(속성) : 클래스 내에 속한 객체들이 가지고 있는 데이터 값들을 단위별로 정의하는 것. (클래스 속 변수)
Method(매서드) : 객체에 정의된 연산을 의미 (클래스 속 함수)

*Class : 공통된 속성의 객체들을 하나의 집합으로 묶은 단위

*Message: 객체들 간 상호작용을 하는 데 사용되는 수단. (method 호출)

*캡슐화(Encapsulation) : 속성과 메서드를 하나로 묶는 것, 객체의 세부내용이 외부에 은폐되어 변경이 발생할 때 오류의 파급 효과가 적음. 재사용이 용이함.
*정보은닉 : 캡슐화에서 중요한 개념, 다른 객체에게 자신의 정보를 숨기고 자신의 메서ㅇ만을 통하여 접근을 허용하는 것. 
1. private |외부에서 값 접근 못하게 막음. ex)private int balance; => balance int속성에 대해 객체 속 메서드로만 접근이 가능.
2. getter  |안전하게 값을 읽음. ex)public int getbalance() => balance 값을 return;
3. setter  |조건 검사후 값을 바꿈. ex)public void setBalance(int balance) => balance가 0보다 클 때 this.balance = balance;

*추상화(Abstraction) : 불필요한 부분을 생략하고 객체의 속성 중 가장 중요한 것에만 중점을 두어 모델화 하는 것. => 그냥 복잡한 걸 단순화시킨거

*상속성(Inheritance): 이미 정의된 부모 클래스의 모든 속성과 연산을 하위 클래스가 물려받는 것이다.
상속성을 이용하면 자신의 클래스에 다시 정의하지 않아도 즉시 자신의 속성으로 부모 속성을 사용할 수 있음.
*다형성(Polymorphism): 같은 메서드명을 가지고 있어도, 객체의 종류에 따라 다르게 실행되는 것.
1. Overloading : 같은 이름의 메서드가 매개변수의 타입이나 개수에 따라 다르게 작동.
2. Overriding : 부모 클래스에서 정의된 메서드를 자식 클래스에서 재정의하여 서로 다른 작동을 구현.
