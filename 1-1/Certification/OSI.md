**OSI 7계층**
*국제 표준화 기구 ISO에서 개발한 네트워크 계층 표현 모델
각 계층은 서로 독립적이며, 하위계층일수록 기계에, 상위 계층일수록 소프트웨어에 가깝다.
하위 계층 : 데이터를 목적지에 정확하게 전달하는 것에 초점을 둠.
상위 계층 : 송수신 데이터의 가공 및 활용에 초점을 둠.

**7계층**(물데네전세표응)
하위 ) 물리 -> 데이터 -> 네트워크
상위 ) 전송 -> 세션 -> 표현 -> 응용

(비프패세대)
*물리계층 : 실제 장비들을 연결하기 위한 연결 장치. 전송단위 : Bit  ==> 물리적 매개체를 통해 데이터를 전송하는 담
*데이터 링크 계층 : 오류와 흐름 제어, 신뢰성 있는 데이터 전송. 전송단위 : Frame   => 연결된 노드간 데이터 통신 담당
*네트워크 계층 : 목적지 호스트로 데이터 전송, 단말 간 데이터 전송을 위한 최적화된 경로 제공. 전송단위 : Packet   => 최적경로담당
*전송 계층 : 네트워크상에서 End to End 형태로 송수신 간 신뢰성 있는 통신 보장. Segment or Datagram (TCP or UDP) => 신뢰담당
이떄 TCP는 신뢰성이 있고, UDP는 신뢰성X(에러검색 X) 대신 빠름.
*세션 계층 : 송/수신간 연결 접속 및 동기제어 : Data
*표현 계층 : 데이터 형식 설정, 부호교환, 암/복호화 : Data
*응용 계층 : 사용자 환경 제공(이메일,웹) : Data

**계층별장치**
*물리 계층
허브 : 가까운 컴퓨터를 연결하여 네트워크로 보내거나 수신한 정보를 여러대의 컴퓨터로 송신하기 위한 장치
리피터 : 디지털 신호 증폭 (증폭기 또는 재생기라고도 함)

*데이터 링크 계층
브리지 : 같은 프로토콜(통신 규약) 간의 두 시스템을 연결하는 네트워킹 장치, 두개의 LAN을 연결하여 더 큰 LAN을 만든다.
느리고, 소프트웨어 방식으로 처리하기에, 포트들이 제한적이지만 서로 다른 속도로 연결할 수 있다.
스위치 : 전송 중 패킷 충돌이 일어나지 않도록 MAC 주소 테이블을 이용해 목적지 MAC주소를 가진 장비 측 포트로만 프레임을
전송하는 장비, 제공하는 포트들이 브리지보다 많으며 하드웨어 방식으로 처리하기에, 
속도가 빠르며 각기 다른 속도를 지원하도록 제어 가능.

*네트워크 계층 
라우터 : 패킷의 위치를 추출, 그 위치에 대한 최적의 경로를 지정 이 경로에 따라 데이터 패킷을 다음 장치로 전향시키는 장치.

*전송 계층
게이트웨이 : 서로 다른 네트워크로 이동하기 위한 관문, 프로토콜 구조가 전혀 다른 네트워크의 연결을 수행.
데이터 형식 변환, 주소 변환, 프로토콜 변환 등을 수행하여 라우터보다 포괄적인 개념.

**정리**
네트워크 통신을 위해서는 참여자들 사이의 약속된 통신 방법이 필요함. 이것이 프로토콜인데 다양한 기능을 하나의 프로토콜만으로 해결할 수 없기에, 
모듈화를 진행함. 각 기능이 계층적으로 기능한다 해서 만들어진게 OSI 7계층, 상위계층은 하위계층의 프로토콜이 제공하는 기능을 사용하여 동작함.
*응용계층은 HTTP, DNS, SMTP, FTP 프로토콜을 (각각 어플리케이션 목적에 따라 다른 프로토콜)
*표현 계층은 어플리케이션간 통신에서 메시지 포맷을 관리함.
*세션계층은 어플리케이션 간 통신에서 세션을 관리함. (RPC) 
*어플리케이션 간 통신 담당, 목적지 어플리케이션으로 데이터 전송 TCP(안정적이고 신뢰성있는 정보 전달), UDP (필수 기능만 제공)

즉 결국 물리적인 광섬유 케이블이나 라우터 위에 프로토콜을 얹어서 데이터 통신이 가능해졌다.
